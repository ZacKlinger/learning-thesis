## Blocker for next run

The fetcher-vs-publisher problem has a long tail. Three failure modes
are now distinct:

1. **Truly failed fetches** (curl 4xx/5xx) — handled by retrying with
   a different URL on the wishlist. Self-correcting.
2. **Landing-page-masquerading-as-PDF** — curl returns HTTP 200 but
   the bytes are HTML. The current workflow does not detect this. *Worth
   considering a content-type / file-magic check in the workflow.* For
   now, the agent inspects on the next read and demotes manually.
3. **Paywalled hosts the agent cannot route around** — NYT, Tandfonline
   (Hmelo-Silver), most journal article pages. These will likely require
   Zack to supply the file directly.

content-type validation step to the workflow, and (b) which of the
