# sources-wishlist.txt
#
# Driver for the fetch-sources GitHub Actions workflow.
# Each non-blank, non-comment line: <filename> <url>
# The workflow downloads <url> to sources-raw/<filename> if missing, commits
# the result, and pushes. The agent then reads sources-raw/ on the next run.
#
# Format rules:
#   - Filename must not contain a slash (sandboxed to sources-raw/).
#   - Suggested filename convention: AUTHOR-YEAR-shortname.ext
#   - Prefer URLs that return the raw bytes (PDF, plain text, HTML), not pages
#     that require JavaScript to render content.
#   - URLs that have rotted will be logged in the Actions run; the agent
#     should demote them to a leads/ note and find an alternate mirror.
#
# Failure modes the agent has observed (2026-05-23):
#   - curl --fail accepts an HTML landing page returned with HTTP 200 as a
#     successful fetch (Springer link.springer.com, AEAweb aeaweb.org). The
#     bytes are not the PDF — the page has a JS-rendered download button.
#     For these, the line is COMMENTED OUT below and the source is demoted
#     to leads/ pending a working raw-PDF mirror.
#   - Hosts behind a cookie wall, paywall, or aggressive bot detection
#     (NYT, Taylor & Francis on Hmelo-Silver, bfskinner.org on the Skinner
#     1958/1954 papers) appear to have returned 4xx and were logged as
#     FAILED in the Actions run.
#
